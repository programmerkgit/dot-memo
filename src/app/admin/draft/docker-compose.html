ネットワークモデル

Dockerのネットワークを考えるとき、ホストマシーン(Dockerを動かしている本体のマシン)のネットワークと、Dockerのネットワークを区別することが重要です。

ホストのネットワークとDockerのネットワークはそれぞれ独立しており、デフォルトではお互いにアクセスできません。

ホストマシンのネットワークからコンテナのネットワークにアクセスするには、 --publishなどのオプションを与えることでお互いを接続させます。

--publishオプジョン

--publishオプションは、ホストマシーンのネットワークのportと, Dockerのネットワークのportを接続します。 docker run --publish=3000:3000コマンドによって、ホスト側のport3000とコンテナのport3000が接続されます。 ブラウザからlocalhost:3000に接続すると、コンテナに接続できます。
