記憶階層

全てのプログラムを主記憶に置くことが理想ですが、それでは費用が非常に高価になってしまいます。

費用と性能の問題に対処するため、以下のような記憶階層が考えられました。

  1. キャッシュメモリ
  2. メインメモリ
  3. 2次記憶

上のもの程アクセスが高速ですが、高価なものになります。

キャッシュメモリは高価ですが、参照の局所性という性質によって、小さな容量でも十分な効果を発揮することがわかっています。

参照の局所性とは、頻繁にアクセスされるデータが全体の一部に偏るという性質です。

記憶管理技法

記憶階層を管理するための仕組みが記憶管理技法です。

記憶管理技法は次の3つから成り立っています。

  1. フェッチ技法(プリフェッチ、要求時フェッチ)
  2. 割付技法
  3. 置き換え技法

フェッチ技法

フェッチ技法には、次の2つの技法があります。

  • 要求時技法
  • プリフェッチ技法

フェッチ技法は、プログラムやデータをいつ主記憶上に配置するかに関するものです。

実行中のプログラムによって参照された時に主記憶に転送するのが要求時フェッチ技法です。 一般に、どのプログラムやデータが参照されるかを参照する前に知ることはできないため、この技法が利用されます。

参照前に主記憶上に転送するのがプリフェッチ技法です。

割付け技法

割付け技法は、読みだしたプログラムやデータを主記憶上のどこに配置するかに関するものです。

置き換え技法

すでに主記憶上にあるプログラムやデータのどの部分を2時記憶に追い出すかに関するものです。

割付技法の種類

メインメモリに対して連続的な区画に割りつけるか、非連続に割り付けるかなどで以下のように分類されます。

以下のように分類されます。

  • 連続割付(contiguous allocation)

      単一ユーザー
      • 単一連続割付(single partition allocation)
      複数ユーザー
      • 固定区画割付(static partition allocation)
      • 可変区画割付(dynamic partition allocation)

  • 非連続割付(noncontiguous allocation)

    • ページング(固定)
    • セグメンテーション(可変)

プロセスとは

プロセスはタスクとも呼ばれます。

CPUのスケジューリングの対象となる基本単位がプロセスです。 プロセスは明確に定義される訳ではないですが、一般的に実行中のプログラムをプロセスと呼びます。

スワッピング

スワッピングとは、優先度の低いプログラムを(2次記憶)に追い出し、優先度の高いプログラムをメインメモリに読み込むことです。

この時利用される2次記憶をバッキングストアと呼びます。

スワッピングは、プロセスの切り替えプロセスを2次記憶を利用して行なっていると考えられます。

この場合、プロセスの実行可能キューに対応する、2次記憶上に存在するプロセスのキューが作成されます。

補助記憶に退避しているプログラムの読み込みをスワップインといい、追い出すことをスワップアウトと言います。

CPUスケジューラは、呼びだすプログラムを指定してディスパッチャを呼び出します。ディスパッチャは実行すべきプログラムがメインメモリ上にあるかどうかを調べ、存在していない場合は不要なプログラムをスワップアウトさせた上で、 必要なプログラムをスワップインさせます。

その後、レジスタを回復して制御を該当のプロセスに移します。
