ブロックヘッダーのデータ構造

ブロックヘッダーの中身は以下の通りです。詳細は後ほど説明していきます。

  • Version: ビットコインのバージョン
  • Prev Block Hash: 前のブロックのハッシュ値
  • Merkle Root: トランザクションの要約情報
  • Timestamp: ブロックの生成時刻
  • Difficulty Target: マイニングの難易度を示す値
  • Nonce: マイニングによって得られた値

ブロックのハッシュ値はブロックヘッダに含まれず、これらのヘッダ情報をハッシュ化(ダブルSHA256)して得られることで計算されます。

特に重要な点は、ブロックヘッダーが前のブロックのハッシュ値を含むことと、トランザクションの要約情報を持つことです。

ヘッダーがトランザクション情報の要約と前のブロックのハッシュ値を含むので、 このハッシュ値はトランザクションの中身が変わっても、前のブロックのハッシュ値が変わっても変わってしまいます。

「ブロックチェーンの基本的な仕組み」のデモで示した通り、 このハッシュ値の依存関係によって全てのブロックがチェーンのように連結されるのでした。

  • Prev Block Hash
  • Merkle Tree
  • Block Height
  • Transactions
  • Difficulty
  • Nonce
