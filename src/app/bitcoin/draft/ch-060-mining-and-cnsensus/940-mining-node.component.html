ブロックの作成

ブロックチェーンに埋め込むブロックはマイナーが生成します

ブロックはトランザクションを集めたものです。

マイナーはネットワークからトランザクションを受け取り、トランザクションプールと呼ばれるメモリ領域にトランザクションを貯めていきます。

トランザクションが集まると、マイナーが構築した新たな空のブロックにトランザクションを含めていきます。

このブロックは新たにブロックチェーンに含まれる候補となるブロックであり、候補ブロックと呼ばれます。

マイナーは候補ブロックのナンスの値を計算し、正解を探します。

マイナーはマイニングをしながらも、次のマイニングのためにトランザクションプールにトランザクションをためていきます。

マイナーが正解を見つけるまでにネットワーク上からブロックを受け取ると、ほかの候補ブロックがマイニングに勝利した事になります。

そうするとマイナーは、そのブロックに含まれているトランザクションをトランザクションプールから削除し、まだブロックに含まれていないトランザクションを集めて新たな空の候補ブロックに含め次のマイニングを開始します。

マイナーが正解を見つけると、ナンス値をブロックに埋め込み、ブロックチェーンにブロックを埋め込みます。

The process of Transaction accumulation

Get Transaction from another node

Put Transactions into transaction pool

construct to candidate block

Start Mining and accumulate transactions for next candidate block

If received new block, remove transactions in the block from transaction pool.

Accumulate transactions and make candidate block again

When Jim found the solution to proof of work, that block is validated
